class xpsi.PostProcessing.Run(filepath, ID, names, bounds, labels=None, implementation=None, truths=None)[source]¶

Base class for a sample object, containing basic information.

Parameters:
  • ID (str) – For identification of the set of the samples if multiple sets are jointly plotted.
  • filepath (str) – Path to samples file.
  • names (list) – An ordered list of str parameter names. The ordering must match the parameter vector ordering defined in sample backend objects.
  • bounds (dict) – A dictionary of one-dimensional hard parameter bounds. See getdist.mcsamples.MCSamples; the keys must match the names list. For the purpose of density estimation plots these bounds can be viewing bounds.
  • labels (list) – An ordered list of (LaTeX compatible) str literal parameter labels.
  • implementation (str) – Sampling software applied to generate the samples. Known options are ['multinest', 'polychord', 'emcee'].
  • truths (dict) – Optional dictionary of parameter truths, if known; if unknown leave as None. The keys must be names which match those in names.

class xpsi.PostProcessing.PostProcessor(runs=None, likelihood=None)[source]¶

Post-process samples for inference and posterior checking.

Calculate inferences, usually in the form of approximate integrals over the posterior distribution. Also visualise posterior information and derived quantities.

Perform posterior predictive checking.

Parameters:
  • runs – An instance of Runs or an instance of Run.
  • likelihood – The instance of Likelihood used for sampling, or a clone of that instance. Defaults to None. If multiple likelihood functions are associated with the runs due to discreteness in the model space, one can pass a dictionary with keys matching the IDs of the runs.
plot_posteriorDensity(self, params, run_IDs=None, combine=False, combine_all=False, only_combined=False, bootstrap_estimators=True, bootstrap_density=False, separate_plots=False, write=False, root_filename='', directory='./', ext='.pdf', dpi=300, maxdots=2000, **kwargs)[source]¶

Generate posterior density plots with getdist and nestcheck for nested sampling runs.

Up to five runs can be plotted natively via nestcheck; beyond such a number the plots are generally display too much information and clarity is lost.

Parameters:
  • params (list) – List of parameter strings for plotting. Must match identifier strings which are attributes of Run instances, and those parameters must occupy the same indices across the Run instances (a limitation, but the nestcheck package assumes a set of realisations of the same nested sampling process).
  • run_IDs (list) – An iterable containing string identifiers of instances of Run instances. These identifiers must match objects collected in the Runs instance stored as an attribute of an instance of the PostProcessing class. Defaults to None, meaning attempt to use all runs in the attribute instance of Runs.
  • combine (bool) – Additionally combine the runs into a single run for overplotting?
  • combine_all (bool) – Combine all runs in the Runs instance or only those for which IDs are provided? Ignored if combine is False.
  • only_combine (bool) – Only plot the combined run? Ignored if combine is False.
  • bootstrap (bool) – Use nestcheck and fgivenx to bootstrap the runs for posterior density error estimation?
  • separate_plots (bool) – Generate a lower-triangle plot with getdist, and a separate error plot with nestcheck (with fgivenx and getdist). If False (default), the diagonal panels of the lower-triangle plot are modified by adding the nestcheck output. Ignored if bootstrap is False.
  • write (bool) – Export the figure?
  • root_filename (str) – Root filename to prepend to automatically generated name. Can be, e.g., a model and/or data set identifier.
  • directory (str) – If None defaults to current directory.
  • ext (str) – File extension for writing. E.g., '.png'.
  • dpi (int) – Dots-per-square-inch settings for exporting plots.
  • kwargs – Keyword arguments for the _plot_density_with_error() and _plot_triangle() methods. Keyword arguments for line properties (width and alpha) for getdist contours and density distributions. If bootstrap and not separate_plots then the density distribution linewidth is set to zero if not explicitly specified with kward lw_1d.
plot_pulse_and_spectrum(self, params, run_IDs=None, combine=False, combine_all=False, only_combined=False, bootstrap_estimators=True, bootstrap_density=False, separate_plots=False, write=False, root_filename='', directory='./', ext='.pdf', dpi=300, maxdots=2000, **kwargs)[source]¶

Plot data and model for posterior checking.

If a combined run is available, only this run is used. Otherwise a plot is generated for each run using the corresponding likelihood object. There may in principle be multiple Pulse instances per likelihood object, but only the first instance is used.

Compute and plot posterior-averaged channel-summed count-rate pulse. The figure contains four panels which share phase as an x-axis:

  • the first (topmost) panel displays the data in joint channel-phase intervals
  • the second panel displays the posterior mean expectation in joint channel-phase intervals
  • the third panel displays the standardised residuals in joint channel-phase intervals
  • the last (bottommost) panel displays the channel-summed pulse as a function of phase for a subset of samples, optionally using fgivenx; the injected pulse is plotted by default if the injection (truth) vector is known.
Parameters:
  • cache (bool) – Cache intermediary model objects to accelerate post-processing?
  • nsamples (int) – Number of samples to use. Equally-weighted samples are generated, thus introducing a additional Monte Carlo noise which is ignored.
  • num_phases (int) – Number of phases to interpolate at on the interval [0,2] cycles.
  • use_fgivenx (bool) – Use fgivenx to plot iso-probability credible interval contours in target-source count rate as a function of phase.
  • random_seed (int) – Seed numpy.random for determinism.
  • kwargs –

    Keyword arguments for the following:

    • _SignalPlot.add_expected_source()
